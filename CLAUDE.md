## What this is

Backend-only REST API for fleet management ("FleetSys"). Express 5 + TypeScript (run directly via `tsx`, no build step) + Prisma 7 over PostgreSQL. Domain is in Spanish; keep new code in Spanish to match.

## Commands

```bash
npm install
npx prisma generate          # REQUIRED before first run — see "Generated client" below
npx prisma migrate dev       # apply/create migrations in dev (migrate deploy in prod)
npm run start:dev            # run with hot reload (tsx watch)
npm start                    # run once
npm run seed                 # load sample data (prisma/seed.ts)
```

No test runner is configured (`npm test` is a placeholder). No linter.

### Environment

`.env` (gitignored) must define `DATABASE_URL` (PostgreSQL). Also honored: `JWT_SECRET` (falls back to a hardcoded default — set it) and `PORT` (default 3000). `prisma.config.ts` is the Prisma 7 config that points at the schema/migrations and reads `DATABASE_URL`.

## Architecture

- **Single Prisma instance, exported from `src/server.ts`.** Every route does `import { prisma } from '../server'`. There is no separate db module — this is the shared client. It uses the **driver adapter** pattern (`@prisma/adapter-pg` wrapping a `pg` Pool), not Prisma's default connection.
- **Generated client lives in `generated/prisma/` (gitignored).** Imported as `../generated/prisma/client`. Nothing runs until `prisma generate` has produced it. Regenerate after any schema change.
- **Routes are the API surface.** One file per resource in `src/routes/`, each an Express `Router` mounted under `/api/<recurso>` in `src/server.ts`. They follow a uniform CRUD-with-try/catch pattern (500 on error, 404 when not found). Prisma model IDs are camelCase (`vehiculoId`, `usuarioId`), so route `:id` params are `Number(id)`.
- **Costs/indicators engine is split from I/O.** `src/services/costos.ts` holds **pure functions** implementing the "macroproceso MA 122 03 01" cost model (CVV, CKV, IUV, depreciation, etc.) with no DB access, so the math is testable in isolation. `src/routes/costos.ts` does the Prisma orchestration around them. Keep new calculation logic pure and in the service; keep persistence in the route.
- **Auth is login-only.** `POST /api/auth/login` verifies with bcrypt and issues a JWT. **There is no auth middleware** — no other route is currently protected. Don't assume `req.user` exists.

## Data model

~27 models in `prisma/schema.prisma` (rendered ERD in `ERD.svg`, auto-generated by `prisma-erd-generator` on `prisma generate`). This is the source of truth for the domain — read it before touching any route.
